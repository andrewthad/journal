# Returning a Fixed-Length Array

I've been writing some code in C recently. One pattern I've found myself
using is allocating a buffer on the stack and then passing it to a function
(as an argument) that fills it and returns void. This only works when the
size of the array is statically known. In most high-level languages, the
callee would just be designed to allocate the buffer itself and return it.
And even in C, this is how the callee would be designed if the size were
unknown.

I wonder if it's possible understand returning a fixed-length array as
a special calling convention. It ultimately needs to be lowered to an
out parameter. I can see a simple way to do this, but it requires a copy
to be performed at the very end. Or maybe we don't need a copy. Normally,
a "copy and freeze" operation would needed, but if we are "returning"
a stack-allocated array, we are actually guaranteed that there are no
aliases.
