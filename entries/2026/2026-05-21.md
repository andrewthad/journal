# Multiply Bits

Writing this out so that I can remember it:

                     1011
    x 0001 0001 0001 0001
    =====================
      1011 1011 1011 1011

The general idea is that it's possible to broadcast a bit pattern by
creating a repeating sequences of the number 1 padded out to the same
length of the original bit pattern.

                00wx 00yz
    x 0001 0001 0001 0001
    ===================== (trucate at 16 bits)
      00yz 00yz 00yz 00yz
      00wx 00wx 00wx
    =====================
              (yz+wx)  yz

# Messing With StreamVByte

If I disable SIMD on the upstream streamvbyte project, my decode rate
plummets from about 8 billion log-width-uniform 32-bit integers per
second down to 200 million.

On a very experimental branch, I have removed the lookup table used
for decoding and replaced it with code. My decode rate is not very good.
I'm at about 3 billion 32-bit integers per second. Using the 256-bit
registers does not help because you cannot shuffle across the two
128-bit lanes.
