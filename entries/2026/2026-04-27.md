# Unifying Dictionaries and Base-Offset Encodings as Tries

A thought that I keep having as I'm worked on compression is "what about a
trie?". Roughly, I'm thinking that there are several data distribution
characteristics worth targeting with compression:

* Small domains, values being near one another, distributions near
  zero (byte count, packet count)
* Repeated values (document id)

There's another thing that gets discussed in the literature: repetitions.
RLE is the one thing that I'm not interested in.

I learned about a data structure called a wavelet tree today. I won't
describe it here. As best I can tell, a wavelet tree works as follows.
It needs to repeatedly partition a data set by its midpoint. This partitioning
happens log2(M) times, where M is the number of distinct elements (i.e. the
size of the alphabet).

In the examples I've found, the number of bits used to represent the alphabet
is never a concern. This is because the "leaf labels" are expected to be small
compared to the size of the rest of the data structure. For my purposes, I
cannot count on this. One way to fix this is to use some other data structure
(like an ordinary trie) to represent the set of distinct elements. This way,
we can recover some compression when all the elements are distinct but are
near one another.

I'm not convinced this will work well. I need to think about it more.
